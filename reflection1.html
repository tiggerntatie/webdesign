Web Design Reflection 1: To Validate or Not?

Think back to your first challenge in this course: among other things, you were asked to identify a website that you admire and another site that you do not. Now that you have had a small amount of experience writing web pages by hand, you will return to those sites to see if they are valid HTML. Here is what you need to do for this reflection:

  • Read this page about why it is important to validate your html
  • Visit the website you said you admired. Check it with the markup validator that you have been using to check your own code.
  • Visit the website you said you hated and run it through the validator too.
  • Create a new web page on the web server and name it reflection01.html. On this new page answer the following questions:
    1. Did either of the websites validate without errors? If not, how many errors did they have?
    2. Do you think you should be validating your own web pages, or is it a waste of your time?
    3. Can you find any other opinions (aside from the page I already asked you to read) on the Internet to support your point of view? Summarize that opinion and provide a link to the original source.
    4. Don't forget: make sure your reflection validates. Feel free to add the validation link/image that you added in Challenge 3.

You can research and respond to this reflection as homework and use class time and/or the X to compose your answer using TextWrangler. You can also use shiftedit.net to create and edit the page from home. This reflection is due on the next X day.
