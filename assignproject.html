Summary

It's time to do your own thing! Imagine a web site and make it happen.

Purpose

People learn best by doing. For the rest of the semester you will be working on building a complete web site with multiple pages. While you do this you will have plenty of time to hone the skills you have developed and pick up new ones at the same time.

Due Date

Your project proposal is due on Tuesday, November 8, 2011.

Assignment

During the first quarter you have worked hard to learn the basics of XHTML for page coding and CSS for page styling. You have even gone face to face with computer languages: Javascript and PHP. Altogether you have spent approximately as much time on this as you would have if you just completed your first full week as a novice web designer. Since you have what is effectively almost a second full week available (if we include x-days), you have enough time to put together a significant web site with several (five or more) linked pages.

Project Ideas

I have collected a short list of general project ideas to help you brainstorm:

  • Make a site for a friend's rock band (real or imaginary)
  • Promote a fake movie, product, or computer game
  • Promote your home town as a travel destination
  • Presidential campaign site for a campaign from long ago (Abraham Lincoln?)
  • Promote a fictional public works project (a subway system for Hanover?)
  • Promote your favorite hobby (taxidermy?)
  • Promote your favorite sport or team (think local)
  • Find a non-profit organization in need of a web site
  • A home page for your church group?
  • Redesign the HHS web site (this time with CSS and HTML!)
  • Promote a fictional hobby (bonsai kittens?)

You have some experience with Javascript and PHP, but not enough to do anything big, so keep that in mind if you start to think about creating a site with dynamic content. There are also lots of places where you can download scripts for free, if you don't want to reinvent the wheel.

Here is what you need to do:

  • Think about the project you would like to attempt. You can work with a partner, but there can be no more than two people on a team. You will have to agree to receive the same grade for the proposal, all weekly updates and the final project itself.
  • Your project will need a name. Actually, two names: one name will be what your visitors will see (for example: “When Pigs Fly!”) and the other will become the directory name on your account where the project will be found (for example: whenpigsfly).
  • If you are working with a partner, one of you will host the site on his/her account. Figure out who this owner will be. Keep in mind that both partners will need to log in to that account to edit files.
  • When you're ready, tell me about the project the project owner and who is on the team.
  • You will need to create a project proposal web page. The proposal will stand as your formal description of what you intend to create (after all, you can't attain your goals if you don't know what they are!):
    1. On the project owner's account, create a sub-directory in the public folder, with the short name that you decided on above (for example: create a sub-directory called whenpigsfly in the public directory).
    2. Create another subdirectory called plan inside the directory you created above (for example: you would have a sub-directory called plan in the public/whenpigsfly/ directory).
    3. Create your proposal file, named index.html inside the plan folder that you created above (for example: you would have a file called public/whenpigsfly/plan/index.html. If you need help with these steps, talk to me!
  • Your proposal should contain the following information:
    1. Project name (long version).
    2. Project name (short version).
    3. Team member names
    4. A clickable link to the landing or home page for your project (for example: when pigs fly). The landing page should be named index.html so it will be loaded automatically if someone points their browser to your project (that is, they don't have to know the name of your page).
    5. A paragraph describing the proposed web site. Don't just say “A web site about art”. There are lots of web sites about art. What makes your web site different?
    6. A paragraph for each page (including the landing page) that you are planning for the site, describing the purpose of the page. Be sure to mention any features of these pages that you do not already know how to create (that is, try to figure out what you will need to learn to do for each page).
  • Your proposal must validate with no errors or warnings (of course!).
