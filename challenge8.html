Introducing PHP

Like the Javascript challenge, this assignment will require you to work with a real programming language: PHP. PHP originally stood for “Personal Home Page” but is now considered to stand for (the much more dignified) “PHP Hypertext Preprocessor”.

A basic PHP program can look like an ordinary HTML page, but the file must be named with a .php as an extension, instead of .html. When the web server is asked to deliver a php file, it first scans the file looking for scripting commands and executes them. Once any commands have been executed, the resulting file is returned to the web browser. When the file arrives at the web browser, all of the scripting commands will have been stripped from the page and it will appear to the browser like an ordinary web page.

The Challenge Assignment

This challenge assignment includes a web form (above) with typical information that you might ask of a visitor (for a visitor log, or even a forum). Fill in the form and click the send button to see what happens (don't forget to return here and continue reading).

For your assignment, you just create an ordinary HTML page similar to this one, with a form, and a separate PHP page that processes the form input, stores the form in a database and presents the contents of the database to the visitor in the form of a web page.

As always, this assignment page and the PHP file associated with it are a good example (in function, if not style) of what you will need to do for your challenge. Be sure to study the sources before beginning your own pages.

Before you go copying these examples for your own, willy nilly, be aware that some aspects of the assignment PHP file are very customized to the author's environment. In particular, this piece of code from the beginning of the PHP file: 


$link = mysql_connect('localhost', 'webdesign99', 'frakshus');
if (!$link) {
    die('Could not connect: ' . mysql_error());
}
$db_selected = mysql_select_db('webdesign', $link);
if (!$db_selected) {
    die ('Can\'t use webdesign : ' . mysql_error());
}

In this example, webdesign99 is a mysql username and frakshus is the corresponding password. The name of the database itself is webdesign. If you copy this piece of code you will have to use your own account name and password. Your database name is the same as your account name, but without the two digit number at the end. So, for example, your code might look something like this: 


$link = mysql_connect('localhost', 'hermione13', 'abracada');
if (!$link) {
    die('Could not connect: ' . mysql_error());
}
$db_selected = mysql_select_db('hermione', $link);
if (!$db_selected) {
    die ('Can\'t use hermione : ' . mysql_error());
}

On to the requirements!

  1. Create the following new file on the server: challenge08.html this exact name is important!. Also create challenge08.php and challenge08.css
  2. Your pages (both HTML and PHP) must be styled using the css file you created in the preceding step.
  3. You may personalize your page style any way you like, but the style should be the same for both the html and php pages and this common style must be evident to the visitor (me).
  4. Your page must include a list of web form elements that will be used to generate dynamic content on the php page. Inspect the source of this assignment carefully and use it as an example. The list of form elements should be styled to remove the bullets. Your form should include at least:
    • Three <input> elements with type='text'.
    • Three <input> elements with type='radio'.
    • Three <input> elements with type='checkbox'.
    • One <select> element with at least six <option> elements inside it.
    • One <textarea> element.
  5. The PHP page must use all of the form data in some way, processing it and returning it to the visitor. This assignment example stores form data in a database, but you are not required to do the same. The assignment example displays the product of two numbers that were entered by the visitor and you could expand on this concept in some way.
  6. Your HTML and PHP pages must both validate (both HTML and CSS) and must display the HTML and CSS validation icon/links somewhere on the page.
