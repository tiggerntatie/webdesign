Styling Tables with CSS

Notes on Sizes in CSS

When you work with CSS, you often specify the size of something. With fonts we have done this using the em unit (for example, font-size: 1.1em;), but without any explanation. Here's the explanation and a list of some other units you may use:

em
A unit of length equivalent to the height of the lower case 'm' character. In general, em units are best for styling pages. If you use em to size a font then it signifies how much bigger or smaller your font will be than the default size. For example 2.0em will be twice the height of the default text.
px
The px unit refers to pixels on the reader's screen. Because screen sizes and resolutions vary dramatically for different users, this is a highly unreliable unit of length. It is useful, however, for specifying the finest possible line widths (ex: border: 1px solid black;).
%
The percent unit specifies a size as a percentage of the container. For example, if you say a table will be 50% in width, then it will be half as wide as the web page that contains it.
in
The in unit refers to inches. This is not reliable on computer screens, but may be useful if you are styling a page for printing on paper.
cm
The cm unit refers to centimeters. See the comment on inches, above.

CSS Attributes For Tables

attribute example values used for...
width 75%; Specify the overall width of the table, as a percentage of the page.
border 2px solid #222; Specify a border around the table as a line with width, style and color. The width can be specified using any length units you want, the style is typically solid or dashed and the color can be whatever you choose.
border-spacing 1.5em; Specify the spacing between cells in a table.
caption-side bottom Where will the caption appear? Default is top.

CSS Attributes for Cells, Headings and the Table Caption

attribute example values used for...
text-align left or right or center How will text in the cell be aligned?
vertical-align top or bottom or center How will text in the cell be aligned?
border 2px solid #222; Specify a border around the cell as a line with width, style and color. The width can be specified using any length units you want, the style is typically solid or dashed and the color can be whatever you choose. You can also specify one of the four borders of the cell individually with border-top, border-right, etc.
background #eee; Specify a color or image for the background of the cell.
padding 1.5em; Specify the width of whitespace surrounding the text inside the cell. This may be thought of as a margin for the cell. You can also specify padding-top, padding-right, etc.

Customizing Column Widths

If you want the column widths to be different, you will have to specify the widths individually for each of the <th> tags individually. To do this, specify the class of the tags like this (for example): 


<table>
  <tr>
    <th class="attribute">attribute</th> 
    <th class="example">example values</th> 
    <th class="explain">used for...</th>
  </tr>...

Then in your CSS, you can distinguish between the different <th> tags by specifying their class. Just be sure that all the widths add up to 100%: 


th.attribute, th.example {
  width: 15%;
}

th.explain {
  width: 70%;
}

These are just the tip of the iceberg when it comes to styling tables. See this reference for styling tables for a more exhaustive list.
