Styling Forms and Form Elements

If you follow the advice in the mini-tutorial on HTML forms then you will end up with a fairly ugly looking web form, including bullets on each line (because you should put form elements into a list). Putting form controls into an HTML list may seem like a crazy thing, but it makes some sense. Form controls are inline elements, which means that if you just put a bunch of them into a paragraph then they will string themselves out from left to right, wrapping around at the end of the line. Most web forms aren't arranged like that, as the usual style is to stack the controls on top of each other, or possibly in a couple of columns. An HTML list is a great device for forcing a collection of things to appear in a stacked list, even if you don't want it to be a true bulleted list.

Let's examine some basic styling that we need to make these things look nice:

Styling Containing List

First of all, we don't want this list of controls to look like a bulleted list. For this, we could use this rule in CSS: 


ul.form {
  padding: 0;             /* none of the lefthand whitespace that a list usually has */
  list-style-type: none;  /* no bullets, please */
}

This rule will take effect on every <ul> element with class='form' and leave every other <ul> alone. Padding is set to zero, which will remove the customary left-hand white space of a list. The list-style-type attribute is used with value none to remove the bullets from the list. Now the list just appears like a row of lines without any decoration.

Styling Each Line

Within the list of controls, each line is styled using the following rule: 


ul.form li {
  clear: both;        /* each li is fully below one before */
  padding: 0.2em;     /* space each item in the list */
  margin: 0.2em;      
  overflow: auto;     /* expand to enclose all blocks inside */
  background-color: #fafafa;  /* group each row, visually */
}

Because <li> elements are blocks, they can be styled using float and clear, just like a <div>. The first attribute, clear: both;, forces each line to sit completely below the previous line. Padding and margin are set to enforce a comfortable moat around our form controls. overflow: auto; forces each line in the list to expand vertically to encompass everything it contains. Without this, the contents of each line overflow their lower vertical bounds. The final background-color attribute adds a faint background to each line. The margin set earlier will leave white space around each line's background coloring. You should feel free to experiment with all of these attributes in your own forms.

Styling the Labels

The general pattern for web controls is to have a control label to the left of the control to tell you what to put in the text box (e.g. 'Last Name'). We want all of these to be on the left-hand edge of each line of the list. We can do this by forcing each label that has class='label' to be a block and float it to the left: 


ul.form .label {
  display: block;     /* each label works like a block, */
  float: left;        /* and floated to the left */
  width: 30%;         /* uniform column width */
}

First of all, the selector of this rule is ul.form .label, which means that the rule will affect every element with class='label' that is a child (that is, nested inside) of a ul.form element (an unordered list with class='form'). When selectors are spaced like this, it makes your specification more narrow and precise. The width attribute will make every label with this class fit inside a block that is 30% of the width of the containing block (that is, 30% of the list width).

Styling the Controls

Now that the labels are lined up, we can drop the controls down on their right hand side using this piece of CSS: 


ul.form li .input {
  display: block;    /* each collection of inputs works like a block, */
  float: left;       /* and floated to the left */
}

This is similar to the label styling rule, and it is applied to everything that is defined with class='input' in the HTML. Usually this is applied to any <input> elements, but for some (radio buttons, for example) it is applied to an enclosing <div>. As always, study the assignment 6 source HTML to see the details.
