PHP Scripting Basics

PHP files are just like the HTML files you have been writing, with the single exception that the server actually looks closely at their contents, and makes changes before sending the file off to the web browser. So, for example, you could have a basic PHP page that looks just like this: 


<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
  <link rel="stylesheet" href="http://netdenizen.org/webdesign/screen.css" type="text/css" media="screen" />
  <title>Empty PHP File</title>
</head>
<body>
  <p>Hello, web browser!</p>
</body>
</html>

If you create this file and name it test.html and load it in your browser, it should just display the text, “Hello, web browser!”. If you save this file as test.php and load it in your browser again (don't forget to type in the URL with the php at the end!) it will look exactly the same. So what's the big deal?

The big deal is that when the file is saved with the .php at the end, the web server will allow you to write programs inside the page. For example: 


<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
  <link rel="stylesheet" href="http://netdenizen.org/webdesign/screen.css" type="text/css" media="screen" />
  <title>Empty PHP File</title>
</head>
<body>
  <p>
    <?php print("Hello, web browser!\n"); ?>
  </p>
</body>
</html>

You can see an example of the preceding file with the modification. Notice that instead of just putting the text: Hello, web browser! in the HTML directly it is set inside a special pair of tags: <?php and ?>. The text isn't just placed inside, either. It is presented inside a function call: print("hello, web browser!");. Now try looking at the source of the page. Does it look like the original example? Where did the extra php tag and print stuff go?

When the web server sees the matching <?php and ?> tags, it signifies PHP code. The server will execute this code as if it were a computer program and if there are any print function calls, it will emit new text into the file, as if it were always there. In this way, the web server can generate a file "on the fly" that looks exactly like an HTML file that you created using Textwrangler.

Now It's Your Turn

Create your own version of the file test.php and try the following additional pieces of code inside the <?php and ?>: 


<?php
  $greeting = "courageous web design student";
  print ("Hello, $greeting!\n");
?>

Try saving this to your desktop first, and drag it into your web browser. If should display all of the ugly PHP code because there was no web server to process the code. Now upload it to the web server and load the URL into your browser. It should display a message like this: 

Hello, courageous web design student!

Note that the \n in the printed string tells the web server to insert a "new line" in the text that it emits. This is the same as when you press the return key on your keyboard while you are editing an HTML file.

Variables in PHP are always named with a dollar sign at the start (e.g. $greeting). You can use these variables inside of quoted strings (like this example), which is a pretty neat trick.

PHP is also very good at doing basic mathematics. Try something like this: 


<?php
  $greeting = "courageous web design student";
  $pi = 3.141592654;
  $diameter = 35.0;
  $area = $pi * pow(($diameter / 2.0),2); /* pow raises one number to the power of another */
  print ("<p>Hello, $greeting!</p>\n");
  print ("<p>The area of a circle that is $diameter inches across is $area square inches!</p>\n"); 
?>

This was just a super duper quickie intro to PHP scripting. If you would like to learn more, there are many excellent tutorials on the Internet.
