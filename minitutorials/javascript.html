Add Intelligence with Javascript

If you follow the advice in the mini-tutorial on HTML forms and styling them then you should end up with a reasonably attractive web page form.

The example in the Challenge 6 assignment has a couple of buttons at the end of the page that you use to transfer values from the form to the story paragraph. The HTML for those looks like this: 


<input type='button' value='Show Story' onclick='javascript:challenge6.transferWords("madlibs");' class="input" />

and 


<input type='button' value='Hide Story' onclick='javascript:challenge6.hideStory();' class="input" />

In both cases, the value attribute is used to define the text that appears on the button. The onclick attribute tells your browser what it should do when someone clicks on the button.

When someone presses the 'Show Story' button, the browser will activate the Javascript interpreter that is built into the web browser and use it to execute a function called challenge6.transferWords, using the argument "madlibs". If you have copied the code I have provided up to this point, you will find that this button doesn't seem to do anything at all. Now we will change that!

Link in Some Javascript Code

In the beginning you wrote web pages using only HTML. Then you learned how to style the pages by linking your HTML document to a CSS document. Now you need to link your HTML document to a new kind of document that contains Javascript computer software. To do that, add a line like this inside the <head> block of your HTML page: 


<script type="text/javascript" src="challenge06.js"></script>

This will make your web browser load the file challenge06.js alongside the HTML and CSS documents. The contents of this Javascript aren't displayed in your browser, and they don't have an immediate effect on the appearance of your page.

Create your Javascript File

For your challenge06.js you should create a new, empty file in your public directory on the server and copy/paste the entire contents of my Javascript file in to it. Save the file to the web server.

Tweak your Javascript

Look at your Javascript code. Javascript code is a set of instructions that your web browser can follow to do something useful. Most big web pages on the Internet use at least a little Javascript to make them more user friendly or powerful. Many of the fancy navigation features that you find on the web are created "on the fly" using Javascript. Google's Gmail and Docs sites rely very heavily on Javascript: the pages you see on the screen are defined with HTML but Javascript makes them move and react to you. Javascript is a very powerful and fast programming language and we could devote an entire semester long course only to learning the basics of writing Javascript software. You will learn enough to incorporate existing Javascript code into your web page. This is very representative of the way that web designers actually use Javascript in their daily work.

In your newly created Javascript file, you will find near the bottom of the file a section that looks like this: 


  /* transferWords moves the form content to the following paragraph, field by field */
  this.transferWords = function(story_id) {
    this.addWordbyID("first-name", "first-name-dest");
    this.addWordbyID("birthplace", "birthplace-dest");
    this.addWordbyID("birth-month", "birth-month-dest");
    this.addRadioButtonbyName("birth-decade", "birth-decade-dest");
    this.addWordbyID("secret", "secret-dest");
    this.addCheckBoxbyName("leaning", "leaning-dest");
    this.addWordbyID("world-view", "world-view-dest");
    /* display the story */
    this.storyID = story_id;
    document.getElementById(this.storyID).style.display = "block";
  }

This block of Javascript code was written for the sole purpose of transferring data from the Challenge 6 assignment example form controls to the "blanks" in the story paragraph that follows it. Since it is written to work only with that web page, you will probably need to modify this piece of Javascript code to make it work on your page.

Most form controls are transferred by executing code like this: 


this.addWordbyID("first-name", "first-name-dest");

This example transfers the data found in the control with an id="first-name" to a <span> element with id="first-name-dest". Some of the other lines transfer data from radio buttons and check boxes, so make sure you use the right type of line for each control in your form.

You should rename the ids in your own work to make them more meaningful for your form.

The last line of this code snippet actually changes the CSS display attribute for the paragraph to "block", meaning that the paragraph will be displayed as an HTML block.

What is an ID Anyway?

Up until now you have specified your content by using the class='classname' attribute. Class names can be assigned to as many different elements as you want. On the other hand, the id='idname' attribute is used to assign a unique identification to a single element in your web page. So the Javascript code examines the HTML itself to locate the unique controls, retrieves the control values and transfers them to the unique <span> elements in the story paragraph.

If you want to refer to an element with a id="example" in your CSS so it can be styled in a unique way, then you can use the # character before the id. Example: 


div#madlibs {
  display: none;
}

This rule apples to the One Unique <div> element that is assigned an id="madlibs".
