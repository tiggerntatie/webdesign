Forms and Form Elements

The basic layout of an HTML form looks like this (taken from Challenge 6): 


<form id="visitor-info-form" action="http://www.netdenizen.org/webdesign/formecho.php" method="post">
  <fieldset>
    <legend>Visitor Information</legend>
    <ul class="form">
      <li>
        <label for="first-name" class="label">First Name: </label>
        <input type="text" name="first-name" id="first-name" value="your name" class="input"/>
      </li>
          <li>
            <label for="birthplace" class="label">Your Birthplace: </label>
            <input type="text" name="birthplace" id="birthplace" value="where born" class="input" />
          </li>
...  (more list items and input tags)
    </ul>
  </fieldset>
</form>

All of the form fields shown in this example have an optional 'class' attribute set, which aids with styling the form with CSS.

  • The <form> element encloses all of the data entry elements. The action and method attributes are used to control how the form is uploaded to the server.
  • The <fieldset> element draws an enclosing box border around the form.
  • The <legend> element adds a label to the enclosing box.
  • Every form element is placed inside an unordered list (<ul>). The list is assigned a class (in this case 'form') so that it can be styled in CSS. Each item has:
    • A <label> element that introduces or describes each form field. Its 'for' attribute identifies the id of the input element it is associated with. The class attribute is used to help with styling in CSS.
    • A <input> element that represents a text box, checkbox, radio button or password box. The attribute assignment type="text" indicates that the input element will be a text box that the page visitor can type in. The 'name' attribute is used to identify the entered data when the form is uploaded to the server. The 'id' attribute is used to identify this element to the label element. The 'id' and 'name' attributes are often set to the same value, though this is not required. The 'value' attribute is used create some initial, default text in the box.

Note that the label and input elements are "self closing". They do not have a following </label> tag, but instead end the opening tag with />. There are some additional 'type' attribute values for the <input> element. Some examples of these follow.

  • The password type <input> shows stars when the user types in the box. This is usually used for hiding passwords on forms. 
    
<input type="password" name="secret" id="secret" value="" class="input" />
  • The radio type <input>. Use the same 'name' attribute with several of these in a form and only one of them will be selectable. The value of the 'value' attribute will be uploaded to the server when the user submits the form. 
    
<input type="radio" name="birth-decade" id="birth-decade-70s" value="70s" />
  • The checkbox type <input>. The value of the 'value' attribute will be uploaded to the server when the user submits the form. 
    
<input type="checkbox" name="leaning" id="leaning-liberal" value="liberal" />
  • The button type <input>. The value of the 'value' attribute will be shown as the text on the button. The 'onclick' attribute is used to execute a Javascript program. 
    
<input type='button' value='Show Story' onclick='javascript:challenge6.transferWords("madlibs");' class="input" />

There are a couple of additional form elements worth knowing:

  • The <select> and <option> elements are used to create a pulldown list of selectable items. For example, for selecting a month: 
    
<select name="birth-month" id="birth-month" class="input">
  <option value="January">January</option>
  <option value="February">February</option>
  <option value="March">March</option>
  <option value="April">April</option>
  <option value="May">May</option>
  <option value="June">June</option>
  <option value="July">July</option>
  <option value="August">August</option>
  <option value="September">September</option>
  <option value="October">October</option>
  <option value="November">November</option>
  <option value="December">December</option>
</select>
  • The <textarea> element is used to create a multiline edit box. For example: 
    
<textarea name="world-view" id="world-view" rows="3" cols="30" class="input">What is your world view?</textarea>
    The rows and cols attributes set an initial height and width of the text box, in characters. Note that this element does require an ending tag; the text between them is the value returned to the server.
