float and clear

Text that is grouped in a <div> element pair can be moved or “floated” around the page at will.

This paragraph and the one preceding it are grouped in a pair of <div class='hickory'> elements, with a class of “hickory”. In CSS, this class is styled to float left. The CSS looks like this: 


div.hickory {
  background-color: #eee;
  padding: 0.4em;
  margin: 0.2em;
  width: 20%;
  float: left;
}

The next pair of paragraphs is grouped in another pair of <div class='dickory'> elements and is also floated to the left.

Note that this other block is styled differently and pulls up on the left, next to the previous left floating block. Again, the CSS is here: 


div.dickory {
  background-color: #fdd;
  padding: 0.4em;
  margin: 0.2em;
  border: 2px dotted #000;
  width: 20%;
  float: left;
}

And now a little paragraph that is floated to the right (class='dock'). And its CSS: 


div.dock {
  background-color: #dfd;
  color: #00f;
  padding: 0.4em;
  margin: 0.2em;
  border: 2px dashed #000;
  width: 10%;
  float: right;
}

Any text that you add outside of the floated blocks will be squished into the remaining whitespace of your page. Be sure to look at the HTML source for this page to see how the <div> elements are being used with classes to achieve this effect.

If, instead of using float as a CSS attribute, you use clear, the block will fall below any previous left or right floated blocks. If you use clear: left; then it will fall below any blocks floated to the left. If you use clear: both; then it will fall below any previous floated block. The CSS for this block: 


div.themouse {
  background-color: #dfd;
  color: #00f;
  padding: 0.4em;
  margin: 0.2em;
  border: 2px solid #0f0;
  clear: both;
}
