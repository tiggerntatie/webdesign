How to use <img> tags

The <img> tag is used to insert an image (photograph, graphic, etc.) in your web page. The <img> tag should only be used for adding images that are relevant to the content of the page. Do not use the <img> tag to add images that are incidental to your content. Any image that is part of the style or look and feel of the page should be added using CSS.

For example, if you were writing a page about the battleship Massachusetts, it would make sense to have a photograph of the ship itself. Use the <img> tag for this. If your page also has a background image consisting of an endless vista of white capped waves, to give your page a salty nautical feel, then that image should be added using CSS, because it does not add anything to the information on your page.

One way to understand the distinction is to ask yourself whether a blind person using a screen reading program would need to know anything about the image. If the reader doesn't need to know about the image to understand the article, then it should be added using CSS.

Using the <img> Tag

Following is an example of the <img> in use: 


<img src="pictures/google-150.png" 
     alt="Screenshot of the google.com homepage"
     width="150" height="90" />

There are a few things worth noticing about this:

  • The <img> tag does not have a closing </img> tag. The tag itself is self closing: the ending > is preceded by the / character.
  • The src attribute is used to give URL of the image. If the image is stored in the same directory as the html file then just the image file name is needed. In the example above, the image is google-150.png and it is stored in a directory named pictures. Do not ever use this URL to link to an image on a different web site, as it misrepresents ownership and is effectively stealing the other site's bandwidth.
  • The alt attribute is required with every image. If your reader is blind and reading your page using a screen reader, then they will hear the text that is included in quotations following the alt. This text should describe what is in the picture in a way that would make the picture meaningful to someone who is unable to see it.
  • Always specify the width and height of the image. This helps your reader's browser format the page on the screen before the image has been received from the server.

File Sizes and Formats

There are two image file formats commonly in use today: jpg (or jpeg) and png.

The jpeg format is a compressed image format is known as “lossy”. This means that an image saved in jpeg format does not look exactly like the original. With photographs the difference is not usually noticable unless the compression is extreme. When you use a piece of software like Photoshop to save a jpeg format file, it will ask you for a quality setting, ranging from 1 to 10. Experiment with low quality (which means small file size and high compression) to find a setting that is acceptable visually, while having a small file size. The smaller your image files, the faster your page will load, and the more likely your visitor is to stick around and read or buy something! Note: the jpeg format compression artifacs usually make logos or line drawings look pretty ugly, so use png for them.

The png format is also a compressed image format, but is “lossless”. This means that an image saved in png format does look exactly like the original and has no compression artifacts or visual defects. It does not compress as well as jpeg, so it is not a good choice for photographs. The png format should be used for logos, line drawings or cartoon- like graphics. Any image that you create with solid colors and/or sharp outlines should probably be stored in the png format.

Regardless of which format you choose, be sure to resize and compress the image using a tool like Photoshop before you upload it to the web server.
